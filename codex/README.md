# Codex History Export / Import

Эта папка нужна для переноса локальной истории чатов Codex по текущему проекту между компьютерами и аккаунтами.

Скрипты работают относительно папки `./codex`.

Структура:

```text
./codex/
├── export_codex_history.py
├── import_codex_history.py
├── history/
│   └── codex-history-*.tar.gz
└── README.md
```

## Что сохраняется

Экспортируются только локальные файлы истории Codex:

```text
~/.codex/sessions/**/*.jsonl
```

Не экспортируются:

```text
~/.codex/auth.json
~/.codex/config.toml
токены авторизации
настройки аккаунта
```

То есть переносится только история сессий, а не аккаунт Codex.

## Как работает определение проекта

Скрипты считают, что папка проекта находится на один уровень выше папки `./codex`.

Например:

```text
MTC True Teach Champ 2026/
├── codex/
├── mts-challenge-2025/
└── telegram-history-exporter/
```

В этом случае проектом будет считаться папка:

```text
../
```

относительно `./codex`.

## Экспорт истории

Запускать из корня проекта:

```bash
python3 ./codex/export_codex_history.py
```

Или на Windows:

```bash
python ./codex/export_codex_history.py
```

Скрипт:

1. определит текущий проект как родительскую папку для `./codex`;
2. найдет Codex-сессии, где встречается путь или название проекта;
3. создаст архив в папке:

```text
./codex/history/
```

Пример результата:

```text
./codex/history/codex-history-MTC-True-Teach-Champ-2026-20260625-154500.tar.gz
```

## Импорт истории

На новом компьютере нужно перенести проект вместе с папкой:

```text
./codex/history/
```

После этого запустить из корня проекта:

```bash
python3 ./codex/import_codex_history.py
```

Или на Windows:

```bash
python ./codex/import_codex_history.py
```

Скрипт:

1. найдет последний архив в папке `./codex/history/`;
2. проверит, что в архиве лежат только безопасные файлы истории Codex;
3. распакует `.jsonl`-файлы в локальную папку Codex на новом компьютере:

```text
~/.codex/sessions/
```

После импорта можно попробовать открыть историю:

```bash
codex resume
```

Или открыть конкретную сессию по ID, который покажет скрипт:

```bash
codex resume <session-id>
```

## Где лежат архивы

Все архивы истории хранятся здесь:

```text
./codex/history/
```

Можно хранить несколько архивов. При импорте скрипт автоматически берет самый новый архив по дате изменения файла.

## Повторный импорт

Если сессия уже есть на компьютере, скрипт не перезаписывает ее, а пропускает.

Это сделано безопасно, чтобы случайно не затереть существующую локальную историю.

## Важно

Файлы `.jsonl` могут содержать:

* текст переписки с Codex;
* команды терминала;
* вывод команд;
* пути к файлам;
* куски кода;
* ошибки;
* возможные приватные данные, если они попадали в чат или терминал.

Перед передачей архива другому человеку лучше проверить содержимое.

Посмотреть список файлов в архиве:

```bash
tar -tzf ./codex/history/codex-history-*.tar.gz
```

Посмотреть конкретный `.jsonl` после импорта:

```bash
less ~/.codex/sessions/2026/06/18/example-session.jsonl
```

## Рекомендуемый порядок переноса

На старом компьютере:

```bash
python3 ./codex/export_codex_history.py
```

Затем перенести проект на новый компьютер вместе с папкой:

```text
./codex/history/
```

На новом компьютере:

```bash
python3 ./codex/import_codex_history.py
codex resume
```

## Примечание

Если `codex resume` не увидит импортированную сессию, сами `.jsonl`-файлы все равно будут доступны в:

```text
~/.codex/sessions/
```

Их можно открыть как обычный текстовый файл и вручную перенести важный контекст в новый чат Codex.
