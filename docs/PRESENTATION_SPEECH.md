# Речь к презентации «Научный клубок»

Здесь будет подготовлена речь для выбранных слайдов презентации.

## Слайд 6. Два пайплайна

В основе нашего решения лежат два связанных пайплайна.

Первый превращает документ в граф знаний. Пользователь загружает файл или указывает ссылку, после чего система анализирует не только текст, но также таблицы, изображения и схемы. Языковая модель извлекает сущности, числовые значения и связи, а GraphRAG объединяет их с уже существующими знаниями в Neo4j. Перед публикацией пользователь получает черновик: он может проверить результат, исправить его и только после этого добавить новые данные в общую базу.

Второй пайплайн отвечает за работу с вопросом пользователя. Сначала система определяет тип запроса и при необходимости применяет Compression или Rewrite. Затем выбирается источник информации: внутренний граф либо поиск в интернете через MCP и Yandex Search. Гибридный поиск одновременно учитывает смысл запроса, полнотекстовое совпадение и связи графа. После этого для языковой модели формируется только релевантный контекст, а пользователь получает ответ с цитатами и возможностью перейти к подтверждающему месту в исходном документе.

Все этапы связаны через единый контур сервисов: интерфейс, API, Kafka, GraphRAG и Neo4j.
