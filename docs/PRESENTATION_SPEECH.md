# Речь к презентации «Научный клубок»

Здесь будет подготовлена речь для выбранных слайдов презентации.

## Слайд 6. Два пайплайна

В основе нашего решения лежат два связанных пайплайна.

Первый превращает источник в граф знаний. Пользователь может загрузить документ либо указать ссылку на внешнюю публикацию. Поэтому граф строится не только по внутренним документам, но и по открытым источникам. Система анализирует текст и визуальные данные, извлекает сущности, числовые значения и связи, а GraphRAG объединяет их с уже существующими знаниями в Neo4j. Перед публикацией исследователь получает черновик, может проверить и исправить его и только после этого добавить результат в общую базу.

Второй пайплайн превращает вопрос пользователя в проверяемый ответ. Сначала небольшая локальная модель определяет тип и сложность запроса. Она понимает, достаточно ли обычного поиска или требуется исследовательский режим с более глубоким обходом графа. Для повышения качества поиска эта же модель переформулирует длинные и уточняющие вопросы с помощью Rewrite и Compression. В результате GraphRAG получает самостоятельный и точный запрос, даже если пользователь написал: «А что было во втором эксперименте?».

Далее система решает, достаточно ли внутренней базы. Если пользователь просит найти внешнюю информацию, подключается MCP, который выполняет поиск через Yandex Search. Для внутренней базы используется гибридный поиск. Он одновременно учитывает смысл вопроса, точные совпадения названий, идентификаторов и чисел, связи между сущностями в Neo4j, а также структурные фильтры. Например, можно искать эксперименты только для определённого материала, страны, года или диапазона температур.

В финальную модель передаётся только отобранный контекст. Мы обязываем LLM оставлять ссылки на источники для всех фактов, которые она приводит в ответе. Сами ссылки подсвечиваются разными цветами в зависимости от уровня подтверждённости: факт может подтверждаться одним или несколькими источниками, содержать противоречия либо требовать дополнительной проверки.

При наведении пользователь сразу видит подтверждающую цитату, а при нажатии открывает карточку доказательства. В ней показаны утверждение, точный фрагмент источника, связанные сущности и возможные противоречия. Из карточки можно перейти прямо к нужной странице и конкретному месту документа, из которого был взят факт.

Все этапы работают в едином контуре сервисов: интерфейс, API, Kafka, GraphRAG и Neo4j.
