# Научный клубок

https://www.figma.com/board/6dxuChhjXFamLp5o3b1ago/Pipeline-%D0%BE%D0%B1%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%BA%D0%B8-%D0%B4%D0%BE%D0%BA%D1%83%D0%BC%D0%B5%D0%BD%D1%82%D0%B0?node-id=0-1&t=g4hYfT5gDdOg05QQ-1

«Научный клубок» — поисково-аналитическая система, которая превращает разрозненные научные документы, таблицы и изображения в связанную базу знаний. Пользователь может задавать исследовательские вопросы обычным языком и получать обоснованные ответы с цитатами, исходными документами и связанными данными из графа знаний.

## Ключевые функции

### Построение графа знаний из документов

Система анализирует загруженные научные документы и извлекает из них материалы, эксперименты, процессы, оборудование, свойства, исследовательские команды, выводы и связи между ними. Новые знания объединяются с уже существующим графом в Neo4j, а синонимы и дубли сопоставляются между собой.

### Анализ текста и визуальных данных

Обрабатывается не только обычный текст, но и таблицы, изображения, графики и схемы из PDF, DOCX, PPTX, XLSX и CSV. Это позволяет использовать сведения, которые при классическом текстовом поиске остались бы незамеченными.

### Исследовательский чат с GraphRAG

Пользователь задаёт вопрос обычным языком, а система выполняет гибридный поиск по документам, сущностям и связям графа. В итоговый запрос языковой модели передаётся только релевантный контекст, поэтому ответ опирается на внутреннюю базу знаний, а не на предположения модели.

### Проверяемые ответы

Исследовательские утверждения сопровождаются цитатами непосредственно в тексте ответа. Пользователь может открыть подтверждающий фрагмент, исходный документ и связанные сущности, а также посмотреть финальный запрос, на основании которого языковая модель сформировала ответ.

### Обнаружение проблем в данных

Система автоматически находит противоречия, недостающие параметры, недостаточный объём выборки, расхождения между источниками и другие проблемы качества данных. Обнаруженные проблемы сохраняются как связанные элементы базы знаний и доступны для дальнейшего анализа.

### Поиск в открытых источниках

Пользователь может явно включить интернет-поиск или попросить выполнить его непосредственно в сообщении. MCP Web Search получает сведения через Yandex Search API, после чего языковая модель формирует ответ с указанием использованных внешних источников.

## Дополнительные функции

- Ответ формируется и отображается в реальном времени.
- История этапов обработки показывает, что система делает с запросом и сколько времени занял каждый этап.
- `CompressionQueryTransformer` преобразует уточняющие вопросы в самостоятельные запросы с учётом истории диалога.
- `RewriteQueryTransformer` оптимизирует длинные пользовательские запросы для поиска по GraphRAG.
- Через символ `@` в контекст чата можно явно добавить материал, документ, эксперимент, проблему или другую сущность графа.
- Граф знаний можно исследовать визуально, переходя между связанными сущностями и исходными документами.
- Материалы, эксперименты, документы и проблемы в данных связаны общей навигацией.
- Перед публикацией извлечённых знаний пользователь может проверить черновик результата обработки документа.
- Для обрабатываемого документа отображаются статус, подробный прогресс и прошедшее время; обработку можно отменить.
- Оригинальный документ сохраняется и доступен для скачивания.
- Результат извлечения и найденные проблемы можно скопировать в формате JSON для анализа и отладки.
- Интерфейс поддерживает Markdown, таблицы и форматирование ответов языковой модели.
- Для ответа отображаются время обработки, использованные токены и источники.
